# AGENTS.md — 心晴岛 APP Codex 构建总指令

## 0. 项目身份

你正在构建《心晴岛 APP：校园情绪补给与五育成长伙伴》。它是面向中小学生的心理健康辅助 APP 原型，目标是让学生愿意打开、愿意使用，并在使用中完成情绪觉察、即时调节、行动记录、同伴支持和主动求助。

本项目用于“心理与智能实践方向”参赛作品。作品形式是虚拟作品 APP，必须体现：

- 信息科技与人工智能思维：规则型推荐引擎、风险词识别、本地数据记录、互动式流程。
- 心理学原理：情绪命名、情绪量化、ABC 认知模型、认知重评、放松训练、行为激活、社会支持、自我效能。
- 五育并举：德、智、体、美、劳五类行动卡。
- 科学边界：不诊断、不治疗、不替代老师/家长/心理咨询/医疗服务。

## 1. 技术栈

首选实现为移动端优先的 PWA，并可用 Capacitor 打包为 Android APP：

- React + TypeScript + Vite
- CSS 原生设计系统，不依赖重型 UI 库
- LocalStorage 本地存储，不接入真实服务器
- Vitest 做规则引擎和安全逻辑单元测试
- Playwright 做端到端主流程测试
- Capacitor 作为后续 APK 打包层

不要引入复杂后端、真实聊天、真实陌生人社交、真实心理诊断模型。比赛 MVP 以“可演示、可解释、可测试、可提交”为第一优先级。

## 2. 产品原则

1. 适用性第一，心理学第二：用户看到的是天气、小岛、卡牌、补给、贴纸；评委看到的是心理学原理。
2. 30 秒内得到结果：首次使用必须能快速选择心情、得到补给建议。
3. 每个页面只解决一个明确问题：不要把问卷、文章、图表堆在一起。
4. 弱社交，强支持：允许预设贴纸、漂流瓶、小队树，不开放陌生人自由评论和私信。
5. 不制造比较：不做情绪排行榜、积极排行榜、焦虑排行、连续打卡惩罚。
6. 不诊断：禁止输出“你有抑郁症/焦虑症/心理疾病”等判断。
7. 强情绪有出口：强度 9-10 或风险词命中时，必须引导用户联系真实可信任的人。
8. 所有交互都要有正向反馈：记录、完成行动、主动求助都算成长。

## 3. UI 方向

整体视觉关键词：温暖、校园、岛屿、天气、柔和、卡牌感、轻游戏化。

- 主视觉：心晴岛、天气云朵、阳光、水滴、灯塔、贝壳、小树、小动物。
- 色彩：晨雾蓝、薄荷绿、日光黄、珊瑚橙、薰衣草紫、雾白。
- 风格：圆角大卡片、柔和渐变、玻璃拟态、轻微浮动动画。
- 移动端优先：375px 宽度下必须美观；桌面端以手机壳居中展示。

详细设计见 `docs/03_UI_DESIGN_SYSTEM.md`。

## 4. 页面范围

MVP 必须有以下页面：

1. Onboarding：昵称、最近困扰、支持人设置。
2. Home / 心晴岛：首页、岛屿状态、三个核心入口。
3. Mood / 心情天气：情绪选择、强度、场景、事件输入。
4. AI Guide / 晴晴助手：ABC 引导、换个角度。
5. Action Card / 心晴行动卡：五育任务、完成后再评分。
6. Relief / 情绪急救包：一分钟呼吸、生气暂停、三分钟启动。
7. Social / 温暖小队：漂流瓶、抱抱贴纸、小队树、班级能量树，使用模拟数据。
8. Growth / 心晴册：记录列表、情绪变化、技能卡。
9. Lighthouse / 求助灯塔：支持人、求助话术、风险提示。

## 5. 推荐引擎规则

核心文件：

- `src/services/ruleEngine.ts`
- `src/services/safety.ts`
- `src/data/taskLibrary.ts`

要求：

- 输入：情绪类型、强度、场景、事件文本、ABC 文本。
- 处理：风险词检测、强度分级、场景关键词匹配、五育任务匹配、认知换镜模板。
- 输出：心理小解释、换个想法、五育任务、建议先做的急救动作、安全提醒。
- 单元测试必须覆盖：考试紧张、人际生气、没动力、强度 9、风险词。

## 6. 安全与伦理

绝不输出：

- “你有抑郁症/焦虑症/人格问题”
- “你不需要告诉任何人”
- “只要用 APP 就能解决”
- “不要联系老师/家长”
- 任何自伤方式细节

强情绪与风险词命中时统一引导：

> 你现在需要真实的人陪伴和帮助。请立刻联系身边可信任的大人、老师或家长。如果情况紧急，请联系当地紧急救助渠道。

## 7. 代码规范

- 使用 TypeScript 严格类型，不使用 `any`，除非有注释说明。
- 组件保持小而清楚；页面组件负责布局，业务规则放 `services/`。
- 文案集中、温和、学生化，不使用医疗诊断语言。
- 样式优先使用 CSS 变量，避免内联样式堆叠。
- 不引入真实网络请求；社交数据使用 `mockSocial.ts`。
- 所有新功能都要更新 README 或 docs 中对应说明。

## 8. 测试命令

每次改动后按顺序运行：

```bash
npm run typecheck
npm run test
npm run build
```

涉及主流程、导航、表单、完成行动卡时运行：

```bash
npm run e2e
```

若当前环境无法安装依赖，至少静态检查：

```bash
npm run typecheck
```

## 9. 验收标准

一次完整演示必须走通：

```text
打开 APP → 心晴岛首页 → 我现在很烦 / 心情天气 → 晴晴助手 ABC → 心晴行动卡 → 完成后再评分 → 岛屿变化 → 成长记录 → 温暖贴纸 → 求助灯塔
```

APP 完成后，项目必须能导出：

- 源码压缩包
- 可运行 H5/PWA
- 可选 Android APK
- 页面截图
- 创作日志
- 测试报告
- 用于说明文档的流程图和功能说明
